# Product Requirements — Prototype v1.0

## Product objective

Provide a visual, local system for learning and demonstrating automotive cybersecurity traceability and evidence-based assurance within the APIM, GWM and TCU scope.

## Target user

- Automotive cybersecurity engineer
- TARA analyst
- Security architect
- Verification/validation engineer
- Compliance and assurance analyst
- Supplier cybersecurity reviewer

## Core user outcome

The user can answer:

1. What architecture or trust boundary is being assessed?
2. What asset and damage scenario are involved?
3. What threat, vulnerability and attack path apply?
4. What is the initial risk?
5. What cybersecurity goal and control treat the risk?
6. Which requirements and sources relate to the control?
7. What evidence proves implementation and effectiveness?
8. Is the requirement met, partially met, not met or insufficiently evidenced?
9. What finding and recommendation follow?
10. What residual risk remains after retest?

## Functional requirements

- Clickable APIM/GWM/TCU architecture.
- TARA traceability chain.
- STRIDE and ATT&CK enrichment fields.
- Cross-framework control mapping.
- Evidence repository with file hashing.
- Human assessment form.
- Local guidance engine and optional Ollama integration.
- Findings and remediation tracking.
- Markdown report export.
- Guided and analyst operating modes.
- Seeded interview demonstration scenario.

## Non-functional requirements

- Local-only default network binding.
- No external API requirement.
- No real OEM data.
- No full copyrighted standard text.
- Human approval required for final decisions.
- Clear disclaimer against certification claims.
