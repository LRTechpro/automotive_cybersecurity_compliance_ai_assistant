# Compliance-Assessment Workflow

## Campaign
Defines the exact scope, software baseline, time period, environment, assessor, approver, and exclusions.

## Applicability
Documents whether a requirement applies directly to a component, indirectly to the program, conditionally, or not at all. Every exclusion requires a rationale and review.

## Evidence request plan
Specifies what objective evidence would be sufficient before reviewing artifacts supplied by engineering or a supplier.

## Three-layer evaluation

1. **Design effectiveness:** Would the control satisfy the requirement if implemented as described?
2. **Implementation:** Is the control present and correctly configured in the scoped software baseline?
3. **Operating effectiveness:** Does representative evidence show the control consistently performs as intended?

## Evidence quality
Each evaluation scores evidence from 1–5 for:

- relevance;
- authenticity;
- completeness;
- currency;
- scope.

## Findings and corrective action
A gap is assigned to an owner with root cause, action plan, interim mitigation, target date, completion evidence, retest, and approver.

## Residual risk
The assessor recommends; an authorized owner approves or rejects residual risk. The AI assistant cannot accept risk.
